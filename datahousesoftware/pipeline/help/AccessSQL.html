Access or SQL Server

Pipeline can access data held in either a Microsoft Access data file (.MDB) or in a SQL Server.

An Access file is stored on your PC / laptop, and is accessible by any other computers with whom your PC can share data.

A SQL Server database is stored in a server running MS SQL Server. This can be hosted by you, or by DataHouseSoftware in the cloud.

When Pipeline is initially installed, a MS Access data file is installed on each PC. There are two files,

  • A Test Database for training etc at C:\Pipeline\Data Files\TEST DATA.mdb - this file contains dummy data
  • A blank database ready for your own data at C:\Pipeline\Data Files\LIVE DATA.mdb
    • For several users to access this data simultaneously, the data file must be placed on the network, in a folder accessible to all users. An Access data file is only accessible on the LAN. To back up your data, simply make a copy of the data file (when all users have exited Pipeline). We strongly recomend you use a dedicated backup program such as datamate.

    SQL Server is more robust and can handle thousands of simultaneous users. It can also be configured to allow access over the internet.

    Pipeline can be hosted on a SQL Server hosted in your own office

  • Pipeline must be installed on SQL Server 2008. SQL Server Express is appropraite, free and can be downloaded here http://www.microsoft.com/en-us/download/details.aspx?id=30438
  • This software can be run on desktop operating systems Windows XP, Vista or 7. It’s important to have as much memory as possible.
  • You will need to create one database login for Pipeline users to access the server - you can have several consultants, each with their own PIPELINE username / password all using the same SQL SERVER login
  • You are responsible for backups. We can provide scripts to run daily backups.
  • Occassionally new fields are added to our software, these will need to be added to your database on your SQL Server. In this case either you will need to do it yourself, with instructions from us, or you will need to grant us access to the server to make the changes.
  • To Access the data from outside the office you will need to forward external traffic on port 1433 to your server which is hosting the data. See http://portforward.com/ for details.

    • We can install SQL Server on your PC (remotely if need be) or provide a PC with SQL Server preconfigured. Contact us for details.
