Startup


Pipeline's startup screen - select database and login details.

This is the first screen you'll see after launching Pipeline. There are two functions of this screen.

  • Select a data source (either MS Access data file, OR SQL Server database)
  • Confirm username / password to access the data

  1. Username: This is your username to access Pipeline. It is set by your database admin, and can be changed via the Consultancy section of Pipeline. To access the test data in either Access data file or SQL Server database you can logon with username "pipelinetest"
  2. Password: as above. Set by your database admin via the Consultancy section of Pipeline. For username "pipelinetest" the password is "pipelinetest"
  3. Access (tickbox): Select this if your data is being stored locally on your PC or LAN.
  4. Access Data File: select the location of your Access Data File.
  5. SQL Server: Select this to access data stored in a SQL Server database. Difference between MS Access data file and SQL Server database
  6. Server Login: Each company has one single SQL Server login, with username and password defined by the SQL Server admin. This is the same for each user accessing this database. It is in addition to the username / password for each consultant (see 1 & 2 above). To access test database use login "pipelinetest"
  7. Server Password: Each company has one single SQL Server login, with username and password defined by the SQL Server admin. This is the same for each user accessing this database. It is in addition to the username / password for each consultant (see 1 & 2 above). To access test database use password "pipelinetest"
  8. Driver: Leave as "SQL Server"
  9. Database: The name of your company's database. For test data use "TESTDATA"
  10. Port: Port for SQL Server traffic. Leave as "1433" unless hosting your own SQL Server configured for a different port. For test data use "1433"
  11. Server: IP Address or name of Server. For test data, or data hosted by DataHouseSoftware, please use "119.148.66.102"
  12. update fields... : If required to update fields in your access data file, click this link to run update procedure. This function only works for Access back ends, not for SQL Server. If you are running your own SQL Server, we will supply update queries for you to run. If you are using SQL Server hosted by us in our data centre these updates are run by us.
  13. Relink tables: Automatically ticked if any login details change.
  14. Open Database: Open Pipeline with login details from this screen.
