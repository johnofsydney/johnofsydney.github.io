DataHouse is now free. It is also offered as an MDB file which can be edited in MS Access 2007 or later.

The components are
  • DataHouse - The front end application
  • DataHouse Data Files (aka Back End)
  • Microsoft Access 2007 Runtime




    • Minimum Requirements
    1. Processor: 500 megahertz (MHz) processor or higher (MS recommended minimum)
    2. Memory: 256 megabyte (MB) RAM or higher (MS recommended minimum)
    3. Screen resolution of 1024 x 768 or higher


    Technical Notes
    1. DataHouse is a relational database created using Microsoft Access 2007. The installation file for new users includes Microsoft Access 2007 Runtime.
    2. DataHouse consists of two halves; the data file (back end or server side component) and the application (front end or client side component).
      • The Back End database contains only tables. If there is more than one database user this file needs to be located on a shared network folder accessible to all users
      • The application, commonly referred to in Access documentation as the Front End, is installed to the folder "C:\DataHouseSoftware\DataHouse\" by the installation; each user has a copy on their own PC. This file is launched via the shortcut on the desktop or from the start menu. On opening it connects to the back end data, and contains all of the forms, queries, reports and code that run the database.
      • When using the Access database Back End, the Back end data file and each front end application accessing the data must reside on the same LAN. Wired and Wireless LANs are OK.
    3. Many functions rely on using Microsoft Outlook to send email and to set appointments in the Calendar. Outlook Express (Windows XP) and Windows Mail (Vista) do not provide the required functionality. You need to use Outlook.
    4. Backup of data is important. You should back up the Back End database every night. We use and recommend Data Mate: http://www.datamate.com.au. If you do sign up for their service, please tell them we sent you.
